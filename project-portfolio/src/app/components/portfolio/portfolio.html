Analytics & Data Projects

During my time in the Wisconsin School of Business Master's in Business Analytics program, I had the opportunity to participate in a team analytics challenge. In this project, I worked with two peers to analyze a dataset on hospital readmission and patient demographics.

We were given 90 minutes to complete this task, and we had to prepare a 2.5 minute presentation with our findings to show at the end.

This was a fun project because we had to work together and make compromises in order to extract the best findings and recommendations within the limited timeframe. I often strive for perfection on projects, but I found this experience to be good practice for doing the best possible work in a timebox then moving on, which is often required to in real-world scenarios.

For an honors project in an undergraduate Information Systems course, I created a dashboard in Excel to visualize residential, commercial, and industrial energy pricing across the United States. This project used 2020 data from the U.S. Department of Energy and leveraged pivot tables, slicers, data validation, and VBA macros.

I enjoyed creating this dashboard because it allowed me to utilize geographic tools and combine different features. I also used this project as an opportunity to learn some Visual Basic for Macros, which I found was a powerful way to add interactive features to the dashboard.

During my time in the Wisconsin School of Business Master's in Business Analytics program, I had the opportunity to visit Molson Coors Beverage Company and Second Harvest Food Bank of Southern Wisconsin for career immersions with other students. During both of these trips, I worked with peers on analytics challenges to explore real problems and questions the organizations were facing, and present finding to stakeholders.

At a high level, these projects involved exploratory data analysis, market and industry research, and creating professional visualizations and presentations in a very limited timeframe. I enjoyed these projects since they involved lots of collaboration and because of their potential to impact decisions at real businesses.

For a final project in a graduate cloud technologies course, I created a data pipeline using various AWS services. Raw financial data from S3 buckets was transformed and loaded into a Redshift data warehouse using AWS Glue. From there AWS SageMaker AI was used to build a model that predicted future bankruptcy status based on training data from the data warehouse.

As a final deliverable, I created a presentation for prospective stakeholders that conveyed the architecture, findings, and recommendations in an understandable way for non-technical audiences.

Development Projects

I created this project as a proof-of-concept for a user-friendly interface for an SAP warehouse management system. The main goal was to provide a representative warehouse map and the ability for any user to see accurate inventory levels for every storage bin in the warehouse. I used Blender to build a high-level template for the physical warehouse layout, and the Blender API and Python scripting was used to connect to SAP and transform the template into a detailed model based on the ERP data.

Once the 3D model was built, it was integrated into an existing Angular web application using ThreeJS, and other API connections were implemented to connect live inventory data.

This is one of my favorite projects since I was able to integrate various applications and systems seamlessly in order to make time consuming tasks more intuitive, especially for newer employees. This project also inspired further warehouse planning and optimization efforts at the company, which improved overall efficiency in the long run.

For this project, I worked with a team to create an online storefront for a capstone project in an IT project management course. I was primarily responsible for creating the user interface and API endpoints. The project was fully hosted in AWS and used various services including API Gateway, Lambda functions, EC2, Cognito, and Simple Email Service. Some features included user account creation, carts and a checkout process with order validation, and email notifications.

This project allowed me to learn about integrations between different AWS services, as well as CI/CD and best practices in AWS. It was also my first time working with MongoDB which we used to manage user carts, and I found it to be a powerful tool in specific scenarios.

The portfolio page that you are viewing right now is a simple web app that I built using Angular 21 with PrimeNG components and TailwindCSS. It is being hosted on GitHub pages, and has an automated build and deployment process configured with GitHub actions.
